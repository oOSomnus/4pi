# 整理工作日志

扫描 `.pi/worklog/` 下的每日零散日志，按时间范围汇总成总结文档，原始文件移入归档。

## 流程

### 1. 扫描日志

列出 `.pi/worklog/` 下所有 `.md` 文件（排除 `organize/` 和 `archive/` 子目录）。文件名格式为 `YYYY-MM-DD.md`。

### 2. 确定时间范围

默认按**自然周**（周一至周日）分组。也可按用户指定的其他范围（双周、月）。

识别所有有待整理日志的时间范围。

### 3. 与用户确认

展示将整理的周期列表，例如：

```
发现以下待整理周期：
- 2026-04-20 ~ 2026-04-26（3 个日志文件）
- 2026-04-27 ~ 2026-05-03（5 个日志文件）

确认整理哪些？（回车 = 全部）
```

用户可选择跳过某些周期（如当前未结束的周）。

### 4. 生成总结

对每个周期，读取该范围内所有日志文件。每个文件内可能有多个 `---` 分割的条目（同天多次记录），均应纳入汇总。

文件路径：`.pi/worklog/organize/YYYY-MM/week-YYYY-MM-DD.md`
（`YYYY-MM` 为年月目录，`YYYY-MM-DD` 为该周周一日期）

总结模板：

```markdown
# 周报：YYYY-MM-DD ~ YYYY-MM-DD

## 概览

- 工作天数：N 天
- 主要项目/任务：...
- 整体状态：{on track / delayed / blocked}

## 本周完成

{合并去重后的已完成事项}

## 进度总览

| 任务/模块 | 进度 | 状态 |
|----------|------|------|
| ...      | ...  | ...  |

## 遇到的问题

{汇总本周所有问题，去重合并}

## 关键决策

{汇总本周所有决策}

## 关键数据

{汇总量化指标}

## 下周计划

{汇总所有"下一步"中未完成的事项}

## 备注

{需要长期跟踪的事项}
```

### 5. 归档原始文件

将已整理的原始日志文件移入 `.pi/worklog/archive/YYYY-MM/`，保持原文件名不变。

### 6. 输出结果

```
✅ 已整理 N 个周期：
  - organize/2026-04/week-2026-04-20.md（3 个文件已归档）
  - organize/2026-04/week-2026-04-27.md（5 个文件已归档）
```

## 规则

- **不修改原始文件内容** — 只移动位置
- **去重合并** — 同一事项在多天出现时合并为一条
- **保留关键信息** — 决策、问题、数据不丢失
- **年-月目录** — organize 和 archive 下均按 `YYYY-MM` 分目录
